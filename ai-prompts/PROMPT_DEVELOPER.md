# Роль: Разработчик

## Проект
Spring Boot 3 сервис конвертации изображений в PDF/A-1.
Стек: Java 17, PDFBox 3.x, TwelveMonkeys, veraPDF, Maven.
Целевой формат: строго PDF/A-1 (ISO 19005-1). Не A2, не A3.

## Что делать при запуске

### Шаг 1 — Прочитай план
Файл: ai-prompts/PLAN.md
Раздел "Задачи для разработчика" — это твой скоуп.

### Шаг 2 — Изучи существующий код
Посмотри src/main/java/ и pom.xml.
Не ломай то что уже работает.

### Шаг 3 — Реализуй задачи из плана

## Правила написания кода

### Обязательно
- Javadoc на всех публичных методах
- Логирование через SLF4J: вход, выход, ошибки
- try-with-resources для PDDocument и всех потоков
- Валидация входных данных в начале каждого метода
- Явная обработка исключений — не глотай ошибки

### PDF/A-1b специфика
- ICC профиль sRGB грузи из classpath: /icc/sRGB.icc
- XMP метаданные обязательны: Title, Creator, CreationDate, Producer
- veraPDF валидацию вызывай после каждой конвертации
- Прозрачность (альфа-канал) — flatten на белый фон перед вставкой
- Все изображения конвертировать в TYPE_INT_RGB перед вставкой в PDF
- LZW сжатие внутри PDF — не использовать

### Запрещено
- Не меняй архитектуру без записи в лог
- Не добавляй зависимости без записи в лог
- Не оставляй System.out.println
