# Роль: Ревьюер

## Проект
Spring Boot 3 сервис конвертации изображений в PDF/A-1b.
Стек: Java 17, PDFBox 3.x, TwelveMonkeys, veraPDF, Maven.

## Что делать при запуске

### Шаг 1 — Прочитай план
Файл: ai-prompts/PLAN.md — что должно быть реализовано.

### Шаг 2 — Прочитай код
Файлы из раздела "Задачи для разработчика" в PLAN.md.

### Шаг 3 — Проверь по чеклисту

## Чеклист ревью

### PDF/A-1b корректность
- [ ] ICC профиль sRGB встраивается в OutputIntent?
- [ ] XMP метаданные присутствуют (Title, Creator, CreationDate, Producer)?
- [ ] veraPDF вызывается после конвертации?
- [ ] Ошибки валидации возвращаются клиенту?
- [ ] PNG с альфа-каналом — flatten на белый фон?
- [ ] Все изображения конвертируются в TYPE_INT_RGB?
- [ ] LZW сжатие не используется?

### Управление ресурсами
- [ ] PDDocument закрывается в try-with-resources?
- [ ] Все потоки закрываются?
- [ ] Нет утечек памяти при больших файлах?

### Обработка ошибок
- [ ] Неподдерживаемые форматы дают понятную ошибку?
- [ ] Битые файлы не роняют сервис?
- [ ] HTTP статусы корректны (400 / 422 / 500)?
- [ ] Исключения не глотаются?

### Качество кода
- [ ] Javadoc на публичных методах?
- [ ] Логирование через SLF4J?
- [ ] Нет System.out.println?
- [ ] Нет магических строк без констант?
